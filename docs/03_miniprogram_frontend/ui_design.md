# 微信小程序 UI 设计与交互规范说明书 (基于 9 页设计稿)

本规范说明书基于设计稿中的 9 张微信小程序界面截图进行详细分解，确定了小程序的页面布局、CSS 类名、视觉组件以及交互流程，方便低代码平台、前端开发人员或生成式 AI 直接编写小程序 WXML 与 WXSS。

---

## 🎨 一、 全局视觉体系与 WXSS 规范

### 1. 配色规范 (Theme Colors)
```css
/* app.wxss 全局定义 */
page {
  --primary-color: #38a169;      /* 橄榄绿 (主色调) */
  --accent-orange: #ff7043;      /* 温暖橘 (第二主色) */
  --bg-light: #f7f9fc;           /* 背景淡灰 */
  --card-border: #edf2f7;        /* 卡片边框线 */
  --text-main: #2d3748;          /* 主要文字 */
  --text-muted: #718096;         /* 辅助文字 */
  --status-green: #48bb78;       /* 状态：已完成/通过 */
  --status-red: #f56565;         /* 状态：需补种/警示 */
  --status-orange: #ed8936;      /* 状态：待接种/推荐 */
}
```

### 2. 局部组件通用样式
* **卡片 (.card)**：`background: #ffffff; border-radius: 16rpx; padding: 24rpx; margin-bottom: 20rpx; box-shadow: 0 4rpx 12rpx rgba(0,0,0,0.02); border: 1rpx solid var(--card-border);`
* **圆角按钮 (.btn)**：`border-radius: 40rpx; font-weight: 500; font-size: 28rpx; text-align: center;`
* **状态标签 (.tag)**：`font-size: 20rpx; padding: 6rpx 16rpx; border-radius: 20rpx; display: inline-block;`

## 📱 二、 9 大页面 UI 结构与 WXML 设计

### 1. 首页概览 (Pages/Dashboard/Index)
对应图 1 界面。呈现宝宝档案与今日护理漏斗。

#### 💡 UI 布局结构
* **1. 宝宝信息头部 (Baby Header)**
  * 左侧宝宝头像，右侧名称“小宝贝”，出生日期“2025-02-18 (29w+6 早产)”。
  * 底部双栏显示“实际月龄：5月20天”及“纠正月龄：3月1天”。
* **2. 重要提醒卡片 (Reminder Area)**
  * 红色警告：“下一针补种疫苗 (乙肝疫苗 第2针) 还有 24 天 (2026-07-22)”
  * 橙色警告：“下一次季度儿保 还有 24 天 (2026-07-22)”
  * 蓝色警告：“下一次就诊检查预约 还有 7天 脑炎AC结合疫苗(第1针) (2026-07-11)”
* **3. 今日护理概览 (Daily Goals Gauge)**
  * 四个并排的环形进度条组件（使用 `canvas` 或 CSS `conic-gradient` 绘图）：
    * 饮奶量：`420ml / 500ml` (圆环 84%)
    * 饮水量：`120ml`
    * 排便次数：`2次` (绿色标签)
    * 眼罩佩戴：`2.1h / 4h` (橙色进度条)
* **4. 最近大事记栏 (Milestone Widget)**
  * 紫色提示点，显示最近一条大事记“完成左侧翻身”，卡片带向右箭头 `>` 可点击跳转大事记 Tab。

#### 🛠️ WXML 节点推荐
```xml
<view class="baby-header card">
  <image class="avatar" src="/assets/avatar.png" />
  <view class="info">
    <text class="name">小宝贝</text>
    <text class="birth">出生：2025-02-18 (29w+6 早产)</text>
  </view>
  <view class="age-grid">
    <view class="age-item">实际月龄：<text class="val">5月20天</text></view>
    <view class="age-item">纠正月龄：<text class="val">3月1天</text></view>
  </view>
</view>
```

### 2. 辅食计划与校验 (Pages/MealPlan/Index)
对应图 2 界面。食谱计划与营养规约红绿灯。

#### 💡 UI 布局结构
* **1. 顶部切换栏 (Week Header)**
  * 显示当前周数（如 `第1周 6.30 - 7.6`），两侧带有 `<` 和 `>` 切换箭头。
* **2. 营养校验（本周）Card**
  * 红绿灯形式展示本周辅食合规报告：
    * 主蛋白交替（鱼/虾/鹅肝2-3次）：🟢 2/2 已满足
    * 红肉次数（猪/牛 ≥5次）：🟢 5次 已满足
    * 粗粮占比（≤30%）：🟢 26%
    * 鸡蛋（每日1-2个）：🟢 7天
    * 菜品命名：🟢 规范（无 "+" 号拼接）
    * 排敏/违禁食材：🟢 未发现
* **3. 周历食谱卡片流 (Weekly Meal List)**
  * 纵向排列周一至周日卡片，卡片左边是星期与日期（如 `周一 6.30`），右边分栏展示四餐：
    * 早餐：`蛋黄 + 小米粥`
    * 午餐：`牛肉香菇青菜烩饭`
    * 晚餐：`鳕鱼西兰花南瓜泥`
    * 加餐：`苹果泥`

    * 顶部：日期、事件分类标签（如 `大运动/发育` 蓝色标签）。

* **3. 底部悬浮按钮 (FAB)**
  * 绿色底色大按钮：“+ 记录新事件”。
