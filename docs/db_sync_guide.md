# 🔄 数据导出与微信小程序同步操作指南

本系统在本地运行，为了将您在管理网站上最新规划的辅食周计划、儿保记录、大事记和就诊日志无缝导入到微信小程序，系统设计了一套精致的离线数据导出与同步机制。

---

## 🛑 安全警示（请所有开发者与 AI 助手共同遵守）

> [!WARNING]
> * **严禁修改、覆盖或删除 `docs/miniprogram/db_seeds/` 目录下的任何 JSON 文件**。
> * 该目录中的 JSON 包含用户本人的真实宝宝健康与辅食数据，用于微信小程序在发布和云开发数据库初始化时作为第一手备份种子。
> * 所有从网页端同步生成的新数据，都会写入到新目录 **`docs/db_system_generated/`**，请务必将其区分，防止覆盖真实用户的数据！

---

## ⚙️ 第一步：在管理网站（网页端）编辑辅食与健康数据

在您启动本地管理端服务（运行 `npm run dev` 并访问本地端口，如 `http://localhost:5173/`）后：

1. **辅食计划编辑**：
   - 进入 **“辅食计划”** 页面。
   - 页面支持按周切换。您可以点击 **“➕ 新增下周计划”**，系统会根据最后一周的结束日期自动向后生成 7 天的空白食谱。
   - 点击具体餐别卡片上的 **“✏️ 编辑菜品”**，即可在弹窗内直接以中文顿号（<code>、</code>）、逗号、空格或换行分隔输入本顿吃的所有食材（如“猪肉、青菜、小米粥”），并调整当天鸡蛋目标个数。
   - 保存后，页面将实时触发**红绿灯智能合规性校验**，并将数据写入您的浏览器 LocalStorage 缓存。

2. **其他健康管理**：
   - 您可以进入“身高体重”、“大事记”、“医疗儿保”等页面录入和维护宝宝的成长信息。

---

## ⚡ 第二步：同步生成 JSON 文件

1. 点击左侧导航栏的 **“🔄 小程序同步”** 入口，进入数据备份与同步中心。
2. 在同步中心，您可以直观查看到 8 大核心数据集合在当前 LocalStorage 缓存中的条数与状态：
   - `meal_plans` (周辅食计划)
   - `timeline_events` (成长大事记)
   - `vaccines` (疫苗接种)
   - `healthcares` (季度儿保)
   - `assessments` (发育评估)
   - `clinical_logs` (临床就诊)
   - `safe_foods` (已排敏食材池)
   - `risk_foods` (待排敏食材池)
3. **一键同步保存到项目目录**：
   - 点击底部的 **“一键同步生成至 docs/db_system_generated”** 按钮。
   - 网页将向本地 Vite 开发服务器发起 POST 请求。服务端的保存插件拦截该请求后，会自动在项目根目录下创建 **`docs/db_system_generated/`** 目录，并将 8 个数据集合全部以 <code>YYYY-MM-DD_集合名.json</code> 的格式（例如 <code>2026-06-29_meal_plans.json</code>）自动写入该目录。
4. **手动下载**：
   - 您也可以点击各卡片右下角的 **“📥 下载 JSON”** 按钮，在浏览器端单独下载该集合的标准 JSON 数据包。

---

## 📱 第三步：解锁小程序开发者模式并批量导入

由于小程序代码包不包含您宝宝的私人医疗与生活记录，新用户打开小程序时，界面完全干净脱敏，仅显示“待添加”、“暂无数据”等占位。

如果您想导入您刚刚生成的数据包：

1. 在 **微信开发者工具** 中编译并运行 `wx_miniprogram` 小程序。
2. 在小程序底部导航进入 **“我的”** 页面。
3. 连续快速点击底部的 **“版本号” 10 次**。
4. 伴随手机振动提示，界面将成功解锁并弹出 **🔓 开发者模式** 控制面板。
5. 点击开发者面板上的 **“选取文件并批量导入”**。
6. 在微信文件选取框内，定位到您的项目目录 **`docs/db_system_generated/`**，选中您需要同步的所有 JSON 文件（可多选一并上传）。
7. 小程序的数据导入引擎将自动通过模糊匹配（例如根据文件名中包含的 `meal_plans` 自动将其映射到云数据库对应的 `meal_plans` 集合中），实现一键批量载入！

## 🛠️ 数据转换技术细则说明

为了实现网页 LocalStorage 到微信小程序 JSON 数据格式的无缝衔接，导出模块自动进行了如下转换规范：
* **辅食排餐（meal_plans）**：网页端保存的辅食是 `items` (数组) 格式；在导出至小程序的 JSON 时，程序会自动将其使用 `、` 号拼接，重新组装为小程序端支持的单个 `name` (字符串属性) 格式。
* **脱敏与空数据占位**：当 LocalStorage 中没有任何数据时，同步中心在导出时会加载 `src/data/` 目录下的系统种子数据作为默认备份，使得初次备份也同样拥有健康健全的初始数据结构。
