# 新增辅食计划提示词模版

本模版专门用于指导 AI 助手（如 Antigravity）如何根据已有的**排餐规则**与**已排敏食材池**，快速生成并部署新一周的辅食计划。

---

## 📋 提示词模版 (直接复制发送即可)

```markdown
请帮我生成新一周的辅食计划。具体信息如下：

1. 计划周期：2026年xx月xx日 至 2026年xx月xx日（2026Wxx）
2. 是否有新增已排敏食材：无 / 有（如果是“有”，请列出，例如：新增了“西兰花”和“鳕鱼”）
3. 是否有特殊饮食调整：无 / 有（例如：因为宝宝这两天有点便秘，本周适当减少粗粮比例）

请严格执行以下规则进行编排和输出：
- 严格遵循 [docs/01_已排敏食材池.md] 与 [docs/03_排餐规则.md] 进行排餐，严禁使用未排敏食材。
- 菜品命名需遵循“食材A + 食材B + 食材C + 烹饪方式”规范，严禁使用“+”连接命名。
- 严格执行周频次基线：红肉（猪/牛）≥5次、鱼/虾/鹅肝各2-3次、粥2次、粗面3次、每日1-2个鸡蛋、每餐必须含绿叶菜占位。
- 确认编排后，请为我更新并写入新的周计划文档 `docs/04_周计划-2026Wxx.md`，并同时修改 `src/data/mealPlan.js` 中的导出数据，以便本地开发服务器热更新展示。
```

---

## 🛠️ 后续执行流程说明

当您发送上述提示词后，AI 助手将自动执行以下操作：
1. **食材池校验**：读取 `docs/01_已排敏食材池.md`。如果是新食材，会先追加到该文件中，随后才可被排入计划。
2. **规则推演**：根据 `docs/03_排餐规则.md` 对每天午餐和晚餐的主蛋白、粗粮比例和鸡蛋摄入进行周分配。
3. **文件写入与同步**：
   - 自动生成 `docs/04_周计划-2026Wxx.md`（Wxx 为新的周数），以便您以文档方式浏览。
   - 自动重写 `src/data/mealPlan.js` 数据，使网页端同步更新。
